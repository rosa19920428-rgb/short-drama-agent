# 大纲生成修复说明

## 修复的问题

### 问题1：大纲生成失败
**现象**：点击"生成分集大纲"后进入页面，但没有大纲内容，只有"重新生成"按钮，点击后显示"生成失败"

**原因**：
- 原逻辑中，大纲生成后调用`goToStep(4)`，但step 4是标题选择步骤，不是大纲步骤
- 大纲渲染在step 5，但代码跳转逻辑错误

**修复**：
1. 重构`generateOutline()`函数，移除错误的跳转逻辑
2. 在第5步（大纲页面）添加配置面板和内容面板
3. 生成成功后自动切换显示内容面板
4. 修复错误处理和日志输出

### 问题2：集数选择位置不合理
**原位置**：第1步（项目设置）

**新位置**：第5步（分集大纲页面）

**改动**：
1. 从第1步移除集数和时长选择
2. 在大纲页面添加配置区域，包含：
   - 总集数选择（30/35/40/45/50集）
   - 每集时长选择（60/90/120秒）
   - "AI生成分集大纲"按钮
3. 用户可以在生成大纲前自由调整集数

## 新的工作流程

### 第1步：项目设置
- 选择剧集类型
- 输入项目名称
- ~~集数和时长配置~~（已移除）

### 第2步：创意讨论
- 生成/选择创意（无变化）

### 第3步：角色设计
- 生成/编辑角色（无变化）

### 第4步：标题选择
- 生成/选择标题（无变化）
- 点击"下一步"进入大纲页面

### 第5步：分集大纲 ⭐（重要改动）
1. **首次进入**：显示大纲配置面板
   - 选择集数（默认40集）
   - 选择时长（默认90秒）
   - 点击"AI生成分集大纲"按钮
2. **生成中**：显示loading状态
3. **生成完成**：自动显示大纲内容面板
   - 显示故事弧线可视化
   - 显示详细分集列表
   - 提供"重新生成"和"调整大纲"按钮
4. **重新生成**：点击后清空大纲，回到配置面板

### 第6步：完整剧本
- 生成完整剧本（无变化）

## 测试指南

### 测试大纲生成

1. **正常流程测试**：
   ```
   第1步 → 第2步 → 第3步 → 第4步 → 第5步
   ```

2. **在第5步**：
   - 确认看到"大纲配置"面板
   - 选择不同的集数（如30集）
   - 点击"AI生成分集大纲"
   - 等待生成完成
   - 确认自动切换到内容面板
   - 确认显示正确数量的剧集

3. **重新生成测试**：
   - 点击"重新生成"按钮
   - 确认回到配置面板
   - 可以修改集数
   - 再次生成成功

### 如果还有问题

打开控制台，运行：
```javascript
DebugHelper.checkOutline()
```

查看输出：
- 如果`大纲数组`显示`❌ 不存在`，说明生成失败
- 运行`DebugHelper.showOutlineConfig()`强制显示配置面板
- 运行`DebugHelper.showOutlineDisplay()`强制显示内容面板

## 技术改动详情

### 修改的文件

1. **index.html**
   - 移除第1步的`basic-config`区域（集数和时长）
   - 重构第5步结构，添加两个面板：`outline-config-panel`和`outline-display-panel`

2. **js/ui.js**
   - `generateScript()`：不再从第1页获取集数/时长，使用默认值
   - `generateOutline()`：重写逻辑，添加集数配置获取，移除错误跳转
   - 新增`showOutlineDisplay()`：切换显示内容面板
   - 新增`showOutlineConfig()`：切换显示配置面板
   - `goToStep()`：修改step 5逻辑，根据大纲是否存在显示对应面板
   - `regenerateOutline()`：改为清空大纲并显示配置面板

### 新增的DOM ID

- `outline-config-panel`：配置面板容器
- `outline-display-panel`：内容面板容器
- `outline-episode-count`：集数选择下拉框
- `outline-episode-duration`：时长选择下拉框

## 注意事项

1. **向后兼容**：已有大纲数据的项目可以正常显示内容面板
2. **错误处理**：生成失败时会弹出alert并保留配置面板
3. **灵活配置**：用户可以在生成前多次调整集数
4. **不破坏其他功能**：所有其他步骤（1-4步、6步）保持原有逻辑
