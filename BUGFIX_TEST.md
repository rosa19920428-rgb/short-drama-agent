# Bug修复测试清单

## 🔧 修复内容

### 问题1：所有创意显示为"已选择"
**原因**：AI生成的创意缺少 `id` 字段，导致所有创意的 id 都是 undefined
**修复**：在 parseCreativeIdeas 函数中为每个创意生成唯一ID

### 问题2：创意不能被选择
**原因**：同上，id 为 undefined 导致选择逻辑失效
**修复**：同上

### 问题3：编辑功能不正常
**原因**：编辑后没有正确同步，且选择状态混乱
**修复**：
1. 生成新创意前重置选择状态
2. 确保编辑时保留原始创意的所有字段（包括id）

---

## 🧪 测试步骤

### 测试1：生成创意（2分钟）

1. **刷新页面** (Ctrl+F5)
2. 进入"第一步：创意讨论"
3. 点击"AI生成5个创意"
4. 等待生成完成

**验证点**：
- [ ] 5个创意正常显示
- [ ] **没有**创意显示"✓ 已选择"（应该是全部显示"选择"按钮）
- [ ] 下方**没有**"已选择的创意"区域（因为没有选中任何创意）

**如果看到**：所有创意都显示"✓ 已选择" → 修复未生效

---

### 测试2：选择创意（1分钟）

1. 点击第1个创意的"**选择**"按钮

**验证点**：
- [ ] 第1个创意按钮变为"✓ 已选择"（绿色背景）
- [ ] 其他2-5个创意仍然显示"选择"（金色背景）
- [ ] 下方出现"已选择的创意"区域，显示第1个创意的内容

**如果看到**：所有创意都变成"✓ 已选择" → 修复未生效

---

### 测试3：编辑创意（2分钟）

1. 点击第1个创意的"**编辑**"按钮
2. 修改标题，添加"【已编辑】"
3. 修改外在期待为："测试外在钩子"
4. 修改内在期待为："测试内在钩子"
5. 点击"保存修改"

**验证点**：
- [ ] 显示"✅ 创意已更新"通知
- [ ] 第1个创意显示"✏️已编辑"标记
- [ ] 标题显示"【已编辑】"
- [ ] 核心期待显示修改后的内容
- [ ] 下方"已选择的创意"区域同步更新

**如果看到**：
- 编辑后其他创意变成"✓ 已选择" → 修复未生效
- 编辑后下方区域消失 → 修复未生效

---

### 测试4：重新生成创意（1分钟）

1. 点击"🔄 AI重新生成"按钮
2. 等待新的5个创意生成

**验证点**：
- [ ] 新生成的5个创意显示
- [ ] **所有**创意都显示"选择"按钮（没有"✓ 已选择"）
- [ ] 下方"已选择的创意"区域消失（因为重置了选择）

**如果看到**：新创意自动生成"✓ 已选择"状态 → 修复未生效

---

### 测试5：选择并编辑其他创意（2分钟）

1. 选择第2个创意
2. 编辑第2个创意
3. 保存

**验证点**：
- [ ] 只有第2个创意显示"✓ 已选择"
- [ ] 只有第2个创意显示"✏️已编辑"
- [ ] 下方区域正确显示第2个创意

---

## 🎯 关键指标

修复成功的标志：
1. ✅ 新生成的创意**全部**显示"选择"按钮（没有自动选中）

3. ✅ 编辑创意后，显示"✏️已编辑"标记
