# 创意数量不足诊断指南

## 🚨 问题描述
生成创意时只出现2个（或不足5个），且梗概太短。

## 🔍 快速诊断步骤

### 第一步：查看浏览器控制台日志

1. **打开浏览器控制台**（F12）
2. 切换到 **Console** 标签
3. 清除之前的日志（点击 🚫 图标）
4. 点击"AI生成5个创意"
5. 等待生成完成
6. **查找以下日志**：

#### 查看日志关键词：
```
[AI服务] 解析后的创意数量: X
```
- 如果 X = 2 → 解析问题或AI只返回了2个
- 如果 X = 5 → 正常，但显示可能有问题

```
[AI服务] ⚠️ 警告: 期望生成5个创意，但实际只解析到X个
```
- 如果出现这个 → AI没有生成5个，或解析失败

#### 查看AI原始响应：
```
[AI服务] AI原始响应内容:
创意1：
标题：XXX
...
创意2：
...
```
- 检查是否真的有"创意3："、"创意4："、"创意5："
- 如果没有 → AI偷懒了，只生成了2个
- 检查每个梗概的长度，是否只有1-2句话

---

### 第二步：根据日志判断问题类型

#### 情况A：AI只返回了2个创意
**日志特征**：
- `[AI服务] AI原始响应内容:` 只有创意1和创意2
- 没有创意3、4、5

**原因**：
- AI偷懒了（token不够或理解错误）
- Prompt要求不够明确

**解决**：
1. **刷新页面**（应用最新修复）
2. **再次生成**（每次生成都不同）
3. 如果还是2个 → 点击"AI重新生成"多试几次

#### 情况B：AI返回了5个，但只解析到2个
**日志特征**：
- `[AI服务] AI原始响应内容:` 有创意1-5
- `[AI服务] 解析后的创意数量: 2`
- 有警告日志：`期望生成5个创意，但实际只解析到2个`

**原因**：
- 解析函数bug
- 格式解析错误，把多个创意合并了

**解决**：
1. 复制控制台中的 `[AI服务] AI原始响应内容:` 部分给我
2. 我会根据实际格式修复解析函数

#### 情况C：梗概太短（少于400字）
**日志特征**：
- 5个创意都生成了
- 但每个梗概只有1-2句话

**原因**：
- AI没有遵守字数要求
- Token限制（虽然已增加到4000）

**解决**：
1. 点击"编辑"手动扩充梗概
2. 或者重新生成

## 💡 临时解决方案

如果急需使用，可以：

### 方案1：手动添加创意
1. 点击"手动添加创意"
2. 自己填写5个创意（可以基于AI生成的2个扩充）
3. 确保每个梗概400-500字

### 方案2：编辑现有创意
1. 编辑现有的2个创意，扩充梗概到500字
2. 再手动添加3个创意

### 方案3：多次生成拼凑
1. 生成一次（得到2个）
2. 记下这2个
3. 点击"重新生成"，再得到2个
4. 重复直到有5个满意的
