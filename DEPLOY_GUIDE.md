1. 将整个 `short-drama-script-agent` 文件夹压缩成ZIP
2. 发送给同事
3. 同事解压后，双击 `index.html` 即可在浏览器中打开

**登录信息：**

**⚠️ 缺点：**

- 所有同事共享你的API Key额度

---

### 方法二：部署到免费静态托管（推荐）

将网站发布到互联网，获得一个在线链接分享给同事。

#### 选项A：GitHub Pages（免费，最推荐）

**步骤：**
1. 在GitHub上创建新仓库（例如 `short-drama-agent`）
2. 上传所有文件到仓库
3. 开启GitHub Pages功能：
   - 进入仓库 Settings → Pages
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main" 或 "master"
   - 点击 Save
4. 等待几分钟，会获得链接：`https://你的用户名.github.io/short-drama-agent/`
5. 分享此链接给同事

**优点：**
- 完全免费
- 有在线链接，不用发文件
- 自动HTTPS加密
- 全球访问速度快

**缺点：**
- 代码是公开的（但没关系，你的API Key在代码里所有人都能看到）

---

#### 选项B：Netlify（免费，更专业）

**步骤：**
1. 访问 https://www.netlify.com/
2. 用GitHub账号登录
3. 拖拽整个文件夹到Netlify网站
4. 立即获得在线链接（例如 `https://xxx-xxx.netlify.app`）
5. 可以自定义域名（可选）

**优点：**
- 拖拽即可部署，超级简单
- 自动HTTPS
- 可以设置密码保护（比代码中的密码更安全）

---

#### 选项C：Vercel（免费，适合前端项目）

**步骤：**
1. 访问 https://vercel.com/
2. 导入GitHub仓库
3. 自动部署，获得 `.vercel.app` 链接

## ⚠️ 重要提醒

### 1. API Key额度问题（关键！）

- 当前使用OpenRouter API Key：`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`
- 所有同事共享同一个Key的额度
- 每次生成创意、大纲、剧本都会消耗额度

**建议：**
- **如果同事只是看看**：OK，消耗很少
- **如果同事频繁使用**：建议他们各自注册OpenRouter账号
- **如何查看额度**：访问 https://openrouter.ai/credits

**让同事自己配置API Key（推荐）：**
在 `js/ai-service.js` 第11行：
```javascript
this.API_KEY = '同事的API Key';
```

---

### 2. 数据不互通问题

**现状：**
- 所有数据（创意、角色、大纲）都存在浏览器本地（localStorage/sessionStorage）
- 每个人的浏览器数据独立
- 无法"共享项目"或"协同编辑"

**解决方案：**
- 这是设计意图（保护隐私）
- 如果需要共享，可以导出Word/PDF后发送

---

### 3. 代码中的敏感信息

**当前代码中硬编码的信息：**
- API Key（OpenRouter）
- 访问密码（drama2024）

**风险：**
- 分享给同事 = 他们可以看到并修改这些
- 如果部署到公开网站 = 任何人都可以看到API Key

**建议：**
- 小团队内部使用：风险可控
- 公开部署：建议移除API Key，让每个人自己配置

## 💡 技术细节（给懂技术的同事）

**项目架构：**
- 纯前端静态网站（HTML/CSS/JS）
- 无后端服务器
- 数据存储：浏览器 localStorage/sessionStorage
- AI调用：前端直接调用OpenRouter API

**部署需求：**
- 任何静态文件托管都可以
- 不需要Node.js/PHP等服务器环境
- 不需要数据库

**安全考虑：**
- API Key在前端代码中 = 可被查看
- 密码在前端验证 = 非专业级安全（但够用）
- 无用户系统 = 无法追踪谁做了什么
