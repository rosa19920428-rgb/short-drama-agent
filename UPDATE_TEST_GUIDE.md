# 最新修复测试指南

## 本次更新内容

### 1. 角色名字强制固定
**问题**: AI生成的剧本中，角色名字不稳定，有时会出现中文名
**解决**:
- 修改prompt，添加⚠️ WARNING强制使用已有角色名
- 添加后处理`enforceCharacterNames`，自动检测并替换错误名字

**测试方法**:
1. 生成完整剧本后，随机检查3-5集
2. 确认所有角色名字都与人物小传中的英文名一致
3. 如果出现中文名，告诉我具体的集数和名字

### 2. 添加导出按钮
**功能**: 在完整剧本页面添加Word和PDF导出按钮
**位置**: 剧本页面顶部的工具栏

**测试方法**:
1. 生成完整剧本
2. 点击"📄 导出Word"按钮
3. 检查下载的.doc文件是否包含所有集数
4. 点击"📑 导出PDF"按钮
5. 检查打印预览是否格式正确

## 完整测试流程

### Step 1: 创建项目
- 类型: Female → Billionaire CEO
- 集数: 20集（先小批量测试）

### Step 2: 生成创意和角色
- AI生成创意 → 选择1个
- AI生成角色 → 检查人物名字（Sophia、Alexander等）
- 点击"📖 查看详细小传"确认信息完整

### Step 3: 生成分集大纲
- 点击"生成分集大纲"
- 观察分批提示（1/4批 → 2/4批...）
- 确认20集全部完整

### Step 4: 生成完整剧本
- 点击"生成完整剧本"
- 观察进度（第1/20集 → 第2/20集...）
- 等待约1-2分钟

### Step 5: 检查角色名字
- 随机点开3-5集剧本
- 确认角色名与人物小传一致
- **关键检查**: 女主角应该是Sophia（或其他英文名），不是中文名

### Step 6: 测试导出功能
- 点击"📄 导出Word"
- 检查下载的文件
- 点击"📑 导出PDF"
- 检查打印预览

## 如果还有问题

### 角色名字仍不对
**可能原因**:
- AI忽略了prompt中的警告
- 后处理替换不完整

**临时解决**:
在`js/ai-service.js`第761行，手动修改默认名字：
```javascript
- Protagonist: ${protag?.name || 'Sophia'}
```

1. 按F12打开控制台
2. 点击导出按钮

4. 截图给我
