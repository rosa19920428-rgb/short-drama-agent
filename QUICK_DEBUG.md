# 快速诊断：为什么只显示2个创意

## 立即执行的3个检查

### 检查1：查看控制台日志（最关键）

1. **打开浏览器控制台**（F12）
2. 点击"AI生成5个创意"
3. **找到以下日志**（按顺序）：

```
[创意生成] 即将设置generatedIdeas, 数量: X
[创意生成] 设置完成, generatedIdeas数量: Y
[渲染] generatedIdeas数量: Z
```

**告诉我 X, Y, Z 分别是多少？**

- 如果 X=5, Y=5, Z=2 → 数据在传递过程中丢失
- 如果 X=2, Y=2, Z=2 → AI或解析函数只返回了2个
- 如果 X=5, Y=5, Z=5 → 数据正确但显示有问题

---

### 检查2：查看AI原始响应

在控制台找到这段日志：
```
[AI服务] AI原始响应内容:
创意1：
标题：XXX
概念：XXX
梗概：XXX
...
创意2：
...
```

**检查是否有：**
- ✅ 创意3、创意4、创意5？
- ✅ 每个梗概都很长（多行）？
- ❌ 还是只有创意1和创意2？

**如果只有2个创意**：
→ AI偷懒了，Prompt需要调整

**如果有5个创意但梗概很短**：
→ AI没有遵守字数要求

---

### 检查3：使用诊断工具

1. **打开文件**：`test_parse.html`
   - 文件路径：`/Users/Rosa/short-drama-script-agent/test_parse.html`
   - 直接双击用浏览器打开

2. **复制AI响应**：
   - 在控制台找到 `[AI服务] AI原始响应内容:`
   - 复制从"创意1："到"创意5："的所有内容

3. **粘贴到诊断工具**：
   - 粘贴到文本框
   - 点击"测试解析"
   - **看结果显示几个创意**

**如果诊断工具显示5个，但实际网页显示2个**：
→ UI渲染Bug

**如果诊断工具也显示2个**：
→ 解析函数Bug或AI只返回2个
